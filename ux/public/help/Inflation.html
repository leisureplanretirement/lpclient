How Inflation Works in the Retirement Calculator

The Four Inflation Assumptions

The calculator has four rate assumptions you can set in your scenario. Changing any of these affects every flow using that rate type at once.

Assumption Default What It Controls
annualInflationRate 2.5% General expenses and income tagged as Inflation
annualMedicalInflationRate 4.9% All medical expenses and Medicare premiums
socialSecurityColaRate 2.5% Social Security income growth
marketROR 5.0% Controls how your investment account balances grow over time. It can also be applied to specific income or expense flows that you want to scale with market returns by setting projectionRateType: ROR

How Each Flow Grows Over Time

Every recurring income or expense has a projectionRateType that controls how its amount changes year over year:

Rate Type Meaning
Flat Amount never changes
Inflation Grows at annualInflationRate
ROR Grows at marketROR
COLA Grows at socialSecurityColaRate
MedicalInflation Grows at annualMedicalInflationRate
Manual Grows at a rate you specify per flow via manualProjectionRate (e.g. 0.06 for 6%/year)

Automatic Behavior by Category

You don't need to set projectionRateType manually for medical or Social Security flows — the calculator applies the right rate automatically based on category.

Medical Expenses (category: "Medical")

  • Automatically grow at annualMedicalInflationRate
  • You can override by explicitly setting projectionRateType to Flat, Inflation, or Manual
  • Setting ROR or COLA on a medical flow is not meaningful and falls back to MedicalInflation

Medicare Premiums

  • Projected forward automatically from the most recent Medicare data year using annualMedicalInflationRate
  • No action needed — you do not specify these as cash flows

Social Security Income (category: "SocialSecurity")

  • Set projectionRateType: "COLA" to grow at socialSecurityColaRate
  • You can override with Flat, Inflation, or Manual
  • Setting ROR on an SS flow is not meaningful and falls back to COLA

All Other Flows

  • Default is Flat — no growth unless you explicitly set a projectionRateType

Practical Examples

Scenario How to Configure
Groceries growing with inflation Set projectionRateType: "Inflation" → grows at annualInflationRate (2.5%)
Fixed mortgage payment Set projectionRateType: "Flat" → amount never changes
Medical out-of-pocket costs Set category: "Medical" → automatically grows at annualMedicalInflationRate (4.9%)
Social Security with COLA Set category: "SocialSecurity" and projectionRateType: "COLA" → grows at socialSecurityColaRate
Unusual expense with its own growth rate Set projectionRateType: "Manual" and manualProjectionRate: 0.07 → grows at 7%/year
Key point: The four assumption rates are global dials — changing one affects every flow using that rate type at once. Use Manual with a per-flow rate for expenses that don't fit the global assumptions.
